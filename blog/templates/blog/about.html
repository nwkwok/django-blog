About This Blog

Django

As the title 'Django Blog' implies, this blog was made using Django and was deployed with Heroku. The goal of this project was to try to work through the Django documentation and code from other Django projects/sources to utilize as many Django features as possible.
TL;DR here is my GitHub repo :P

Frontend

The frontend of this blog was structured with templates, styled with Bootsrap while the forms were styled using Crispy Forms. There are some pages that were made using Function-Based Views but, the majority of the views were made with Class-Based Views. If you're unfamiliar with these terms, here's a great article to help outline the differences between the two. This blog uses Django's Paginator to enhance the UX as the number of posts increase.

Backend

While Django comes built-in with SQLite, I configured this blog to use PostreSQL. Profile images are stored in an AWS S3 bucket. User Creation and Authentication are handled using Django's built-in auth forms and user authorization is handled using decorators with the function-based views and mix-ins for the class-based views. Password recovery was configured using Django's built in password reset capability with Gmail as the email server.

Project Dependencies

Here is a list of all the key dependencies used for this project.

  • django... duh :).
  • psycopg2 - to connect with PostgreSQL
  • boto3 - to configure and manage AWS services
  • django-crispy-forms - to style forms
  • bootstrap - to style templates
  • django-heroku - to help configure heroku deployment
